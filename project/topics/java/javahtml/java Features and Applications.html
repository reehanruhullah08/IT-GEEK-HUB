Features of Java


  • Java language was developed by sun-microsystem in 1991 (USA).

  • Java is a platform independent Object Oriented programming language.

  • Different features of java are:-



    • 1. Plattoom Independent :


  • The concept of (write once run anywhere) is known as the platform independent.
  • The program written in one platform can run any platform provided the platform must have the "JVM".
  • Change on Upgrades in OS will not force any changes in Java programmes.



    • 2. Simple, Small and Familiar :


  • Programs are easy to write and debug.
  • Java is a small and simple language because it doesn't have pointer header files , Multiple Inheritance, Operator Overloading.
  • Java code looks like the C++ code and hence Java is a simplified version of C++.



    • 3. Object Oriented


    a. Inheritance:-

  • It is the process of creating the new class from an existing class.

    • b. Polymorphism:-

  • "Poly" means "Many" And "Morphism" means "Form".
  • It means "one name having multiple forms".
  • A single entity shows different characters and behaviors in different circumstances(having the same, but different signature).

    • c. Encapsulation:-

  • It is a mechanism of combining data and functions together in a single unit.

    • d. Dynamic Binding:-

  • It is the way of providing the max. Functionality to a program about the specific type at run time.

    • e. Robust:-

  • Robust means that which is strong, powerful and not affected by others.
  • The compiler checks the program whether there are any errors and the interpreter checks any runtime errors.
  • It makes the system secure from crash.

    • f. Distributed:-

  • Java is a distributed language because the program of Java is compiled on one machine and can be easily transferred to another machine.
  • Java is specifically designed for Internet users which use remote computers.

    • g. Portable:-

  • Java compiler Generates bytecode Instruction that can be implemented on any machine.
  • The feature of "write once run anywhere" makes the Java language portable.
  • Portable means it can transfer from one machine to another machine.

    • h. Dynamic:-

  • While executing the Java programme the user can get the required file dynamically from a computer.

    • i. Secure:-

  • Java ensures that no viruses are communicated with an applet.
  • The bytecode verifier checks the classes after lording.
  • This security becomes important for a language that is used for programming on the internet.

    • j. Performance:-

  • Java is faster than traditional interpretation, since the code is close to native code.
  • Java language has High Performance.

    • h. Multi-threaded:-

  • Multi-threaded means a single program having different threads, executing independently at the same time.
  • Multi-threaded works the same way as multiple processes are run on one computer.

    • i. Compiled and Interpreted:-

  • First Java compiler translates source code into bytecode instructions.
  • Bytecodes are not machine code, In the second stage Java interpreter mach in code that can be directly executed by the machine.

    • j. Architecture-Neutral:-

  • The compiler generates an Architecture-Neutral object file format enabling a Java application to execute anywhere on the network.
  • Then the compiled code is executed on many processes.

    Types of Java Applications


    There are mainly 4 types of applications that can be created using Java programming:



    1) Standalone Application:-


    Standalone applications are also known as desktop applications or window-based applications. These are traditional software that we need to install on every machine. Examples of standalone applications are Media player, antivirus, etc. AWT and Swing are used in Java for creating standalone applications.



    2) Web Application:-


    An application that runs on the server side and creates a dynamic page is called a web application. Currently, Servlet, JSP, Struts, Spring, Hibernate, JSF, etc. technologies are used for creating web applications in Java.



    3) Enterprise Application:-


    An application that is distributed in nature, such as banking applications, etc. is called an enterprise application. It has advantages like high-level security, load balancing, and clustering. In Java, EJB is used for creating enterprise applications.



    4) Mobile Application:-


    An application which is created for mobile devices is called a mobile application. Currently, Android and Java ME are used for creating mobile applications.
